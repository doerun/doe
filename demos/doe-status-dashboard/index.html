Related browser tools
  • Gemma 4 E2B side-by-side demo — WebGPU + CSL simfabric + CSL-WebGPU-emulator lanes with evidence cockpit (identity / speed / accuracy / real-weight badges). Requires the demo server for live CSL runs; static paths work without it.
  • SDK-GUI-style artifact viewer — 6-panel viewer (fabric, PE drilldown, source, host-I/O, trace timeline, evidence overlay). Point it at a compile artifact directory (e.g. bench/out/scratch/gemma4-e2b-csl-sim/compile-L1) to inspect the SDK artifacts + live CSL trace without unpacking SDK binaries. Requires the E2B demo server for its two /api routes.

Serve from the repo root: cd /path/to/doe && python3 -m http.server 8000 then open http://localhost:8000/demos/doe-status-dashboard/. For the side-by-side demo's live CSL runs and the SDK-GUI viewer's /api routes, use python3 demos/gemma4-e2b-csl-sim/server.py --port 8000 instead. Every row is re-derived from JSON at view time; edit the artifact and reload to see the status change.
