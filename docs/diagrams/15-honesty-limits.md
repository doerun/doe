# 15 - Limits: "What this does not prove"

## Purpose

Keep the deck credible. This slide prevents the structural argument from being
read as a performance or full-coverage claim.

## Slide content

- TSIR does not replace performance tuning.
- TSIR is not Doe's only IR and does not cover every CSL kernel yet.
- Simfabric evidence is not hardware evidence.
- A hardware receipt or typed blocker is still required for the Cerebras claim.
- Evidence can be refreshed, but only from a complete Doppler contract.

## Visual spec

- Single strict checklist.
- Use `contrast.gray` for warnings, not Doppler red.
- Use small color chips beside each line:
  - `doe.blue` for compiler scope.
  - `doe.purple` for receipt/evidence scope.
  - `cerebras.orange` for hardware scope.

## Scope guard

- This list is not exhaustive.
- Do not delete a limitation because evidence is inconvenient; move it to slide
  16 when a receipt exists.
